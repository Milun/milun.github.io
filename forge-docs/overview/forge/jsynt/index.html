JSYNT

The content in Forge is built using jsynt, a modified version of the json data format made for Forge and projects like it.

Here is an example of how it looks in the VSCode editor:

Differences to JSON​

If you are familiar with JSON, you'll notice several additional features that JSYNT has.

Multiline strings​

The most useful feature of JSYNT is the support for multiline strings using the <<<"...">>> tags.

Within these tags, you are completely unrestricted. If you are familiar with JSON, here's the features you can use in JSYNT that JSON does not allow in strings:

  • Tab characters
  • Return characters
  • Unescaped \ and " characters

In essence, you can write HTML inside of these tags in the same way you would a normal .html document.

Comments​

Unlike JSON (which usually has no comment support), JSYNT allows for comments using <<<!...!>>> tags. Nothing that is written between these tags will ever be visible to the end user.

Syntax highlighting​

You'll notice that code within the <<<"...">>> tags has HTML syntax highlighting.
