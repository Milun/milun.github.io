Course structure

When developing a course using Forge, you will be working from within the src/course/en/content folder, often referred to as just the course folder for short. Here is an example of the course folder of a simple 3-block Forge unit:

course.jsynt​

This file contains the global settings for your entire course, such as:

articles.jsynt​

An article is a "section" of a page. You can use these to group the "chapters" of your learning material into larger categories.

blocks.jsynt and the b-X folders​

A block refers to a single screen of content. Think of it like a chapter of a book. Each block contains one or more components.

Blocks follow the naming convention of b-X. If you look at the snapshot above, you'll see the folders b-1, b-2, and b-3, which correspond to blocks with the same name.

The blocks.jsynt file is where you specify the order of your blocks. In essence, if you ever need to re-order your content, this is where you would do it. This also means you do not need to re-number your blocks if one of them is removed.

components.jsynt​

These are the main files that you will be working with. Each b-X folder in your course will contain one of these in its root. It's inside of these files are you will define the contents of each block.

b-X/assets​

Any assets (images, videos, downloadable files, etc.) that are used in the b-X block should be included within its own assets folder.

Modularity​

This folder structure is designed in this way to make copying & re-using blocks very straightforward. Simply copy and paste the b-X folder from one course to another to carry everything it needs with it.
