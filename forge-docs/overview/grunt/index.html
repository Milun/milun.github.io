Grunt tasks

Before you can begin development, it's important to understand what 'Grunt tasks' are and how to run them.

'Grunt' is the name of the task-automation library that Adapt uses. Adapt comes with, and I have added, many helper tasks (Grunt tasks) to automate the development process for you. You will need to call some of these tasks yourself, and this is done through the terminal.

While developing, you will have two Grunt tasks running at minimum, grunt dev and grunt server. Please follow the following instructions on how to get those running.

Running Grunt tasks in VSCode.​

  1. With VSCode open, move your cursor to the blue bar at the very bottom of the screen, until it turns into two arrows.
  2. Click and drag your cursor up. This will bring up the terminals window of VSCode.
  3. In the top-right corner of this new window, you should see the cmd icon. If you see a powershell icon instead, please follow the Changing default terminal guide.

You have now opened a Command prompt from within VSCode!

Starting dev mode in VSCode.​

Having done the above, you can do the following to start dev mode.

  1. In the terminal window at the bottom of VSCode, press the 'Split terminal' button (top-right corner, looks like a square with a line down the middle).
  2. You will now have two terminals side-by-side.
  3. In the left one, write grunt dev and press Enter. In the right one, write grunt server and press Enter.

Your developer mode is now running!

warning

The grunt dev task will fail if you haven't added a block yet. See Creating your first course to see how that's done.

Watching grunt dev for errors​

While grunt dev is running, any change you make will be automatically compiled (after a seconds delay), after which point, you can just refresh your web-browser to see the changes applied. You'll know your changes are done compiling when the grunt dev terminal shows a green "Waiting..." message:

However, if you accidentally introduce a syntax error (for example, by forgetting a ,), then the content won't compile until you fix it. When an error has occurred, the "Waiting..." message will not be green (...I couldn't get it to turn red).

When this happens, just scroll up (or increase the terminal window's height) to see what the error was.

If you find yourself refreshing your page and not seeing your changes appearing, it's highly likely you have a syntax error!

Plan B: resetting grunt dev for tough errors​

If you just can't seem to fix an error no matter what you do, you can reset grunt dev to maybe get some more information. To reset it, click on the terminal which is running it, then press Ctrl + C. This will stop the grunt dev task.

Afterwards, type in and run grunt dev again, and it might give you a more detailed error.

tip

If you ever copy + paste an image from one screen to another, you must restart grunt dev. This is because grunt dev will only detect new files if they're made after the last time it updated. Since the image you copied is older, it will never detect it.
